# ClawChat — 产品需求文档 (PRD)

> 一个与 AI Bot 对话的 Web 端应用，以 OpenClaw 为后端示例。

## 1. 产品愿景

打造一个轻量、美观、功能完备的 Web 聊天客户端，让用户可以在浏览器中与 AI Bot（如 OpenClaw）进行单人或多人实时对话，支持 Thread 讨论、命令模式、丰富的消息格式。

## 2. 目标用户

| 角色 | 场景 |
|------|------|
| 个人用户 | 与 Bot 一对一聊天，提问、创作、编程辅助 |
| 团队用户 | 多人在同一房间与 Bot 协作讨论 |
| 开发者 | 通过命令模式快速调用 Bot 能力、调试 |

## 3. 核心功能

### 3.1 聊天模式

- **单人聊天 (DM)** — 用户与 Bot 一对一对话
- **多人聊天 (Room)** — 多个用户加入同一房间，共享与 Bot 的对话上下文
- **Bot 在房间中以参与者身份存在，可被 @mention 触发**

### 3.2 Thread 模式

- 任意消息可展开 Thread（侧边栏）
- Thread 内有独立的对话上下文
- Thread 内可继续 @Bot 获取回复
- 主消息流中显示 Thread 回复计数

### 3.3 命令模式

- 以 `/` 前缀触发命令
- 内置命令：
  - `/help` — 查看可用命令
  - `/model <name>` — 切换模型
  - `/system <prompt>` — 设置系统提示词
  - `/clear` — 清空当前对话
  - `/export` — 导出聊天记录 (JSON/Markdown)
  - `/status` — 查看 Bot 状态
  - `/thread` — 在当前消息开启 Thread
- 命令输入框支持自动补全和提示

### 3.4 消息功能

- **Markdown 渲染** — 支持代码块（语法高亮）、表格、列表等
- **消息编辑** — 编辑已发送的消息，Bot 可重新回复
- **消息删除** — 删除消息
- **Reactions** — Emoji 反应
- **回复引用** — 引用回复特定消息
- **流式输出** — Bot 回复逐字流式展示
- **代码块** — 一键复制代码
- **图片/文件** — 支持上传和预览

### 3.5 用户系统

- **简易登录** — 用户名 + 头像即可进入（MVP 阶段无需密码体系）
- **在线状态** — 显示在线/离线
- **打字提示** — 显示 "xxx is typing..."

### 3.6 UI/UX

- 现代化深色/浅色主题
- 响应式设计，适配桌面和移动端
- 左侧栏：会话列表 / 房间列表
- 中间：消息主区域
- 右侧栏（可收起）：Thread / 用户列表 / 设置
- 快捷键支持（Ctrl+K 搜索, Ctrl+/ 命令, Esc 关闭面板）

## 4. 非功能需求

- **性能** — 首屏加载 < 2s，消息发送 < 100ms 感知延迟
- **实时性** — WebSocket 连接，消息即时推送
- **可扩展** — Bot 后端可替换（OpenClaw 为默认，可适配其他）
- **国际化** — 预留 i18n 结构（MVP 阶段中英双语）

## 5. MVP Scope (v0.1)

| 功能 | 优先级 |
|------|--------|
| 单人与 Bot 聊天 | P0 |
| Markdown 消息渲染 + 代码高亮 | P0 |
| 流式输出 | P0 |
| 命令模式 (/help, /clear, /model) | P0 |
| 多人房间聊天 | P1 |
| Thread 模式 | P1 |
| 消息编辑/删除/Reactions | P1 |
| 用户在线状态 + 打字提示 | P1 |
| 深色/浅色主题切换 | P2 |
| 文件上传 | P2 |
| 导出聊天记录 | P2 |

## 6. 成功指标

- MVP 可在浏览器中完整运行单人聊天 + 命令模式
- Bot 回复流式输出无卡顿
- 移动端可用
